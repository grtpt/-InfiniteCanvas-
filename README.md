二、必须采用的实现原则
1. 所有对象必须存储在统一的数据模型中，不能只画在屏幕上而没有对象数据。
2. 所有对象必须使用“世界坐标”存储，不能直接把屏幕坐标当最终数据。
3. 必须维护视图状态：
   - scale
   - offsetX
   - offsetY
4. 必须提供坐标转换函数：
   - screenToWorld
   - worldToScreen
5. 所有渲染必须基于对象数据重绘，不能依赖一次性绘制后就丢失结构。
6. 所有功能必须在缩放和平移后仍然正确。
7. 撤销 / 重做必须针对对象状态真实生效，而不是只做表面按钮。
8. 导出 PNG 必须基于对象数据离屏重绘，不能截图 DOM。

三、必须满足的关键稳定性要求
以下三项是本任务最关键的硬性要求，必须优先保证：

1. 文字功能必须真实可用
   - 文字模式点击画布后，必须真的出现可编辑输入框
   - 输入后必须能保存成画布文字对象
   - 双击已有文字必须能重新编辑
   - 不能出现“切到文字模式点击没反应”
   - 不能出现“只能新增，不能二次编辑”
   - 不能出现“编辑框位置和缩放错乱”

2. 橡皮擦必须真实可用
   - 必须是真擦除，不是白色画笔
   - 必须使用真实擦除逻辑，例如基于 destination-out 的渲染
   - 导出 PNG 时擦除效果必须仍然存在
   - 不能出现“屏幕上像擦掉了，但导出时又出现”
   - 不能出现“只是覆盖成白色”

3. 缩放时画布不准闪烁
   - 鼠标滚轮快速缩放时，画布不准白闪、不准空白闪、不准重置感闪烁
   - 不能在每次缩放时重建 canvas
   - 不能在每次缩放时重设 canvas.width/canvas.height
   - canvas 物理尺寸只允许在初始化和窗口 resize 时更新
   - 缩放必须通过视图变换和重绘完成，而不是通过反复改 canvas 尺寸造成闪烁
   - 允许使用 requestAnimationFrame 合并重绘，避免频繁闪烁
